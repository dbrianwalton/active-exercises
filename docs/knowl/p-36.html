Paragraph

Attributes:

@name

Name starts with letter and can contain A-Za-z, 0-9 and underscore.

@context

Value is a string specifying how the expression is defined.

context='formula' means the content is a mustache-substitution formula.

context='substitution' means the formula is defined by a variable substitution from another formula. The content contains child elements: #start containing either a de-term defining the initial expression or #var matching a previously-defined expression; #match containing the variable being replaced; and #replace containing another expression defined by #de-term or #var.

I'm wondering about whether it would make more sense for #start and #replace to have a name attribute to match previous formulas and content only if needing to define on the spot, and similarly for #match to only have a name matching the variable and no content.

I can see adding contexts for the derivative of a previously-defined expression. I can also see this allowing non-formula contexts as well, similar to other MathObject contexts (intervals, sets, lists).
