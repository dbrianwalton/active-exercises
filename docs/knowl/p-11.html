My initial idea is that an <exercise> element could have an attribute @dynamic='yes' to indicate that the exercise involves dynamic processing. The PreTeXt processing of such elements would require a comparable strategy for embedding a WeBWorK exercise. This would include generating the corresponding pg files and having a WeBWorK server process the files so that they could be pulled back into the PreTeXt document for final processing. A potential Javascript implementation would need to generate appropriate resources for dynamically inserting the problems into HTML.
